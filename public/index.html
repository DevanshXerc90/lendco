How the Multi-Agent System Operates

🌐

1. Aggregated Context

On call entry, the system pulls data from 5 independent APIs: CRM (e.g. HubSpot), Payments (e.g. Razorpay), Support Tickets, Confluence/Notion KB, and Workflows. It synthesizes an immediate loan status representation.

🔍

2. Gap Analysis (Diagnosis)

Instead of a static script, it dynamically analyzes what data points are missing for the borrower's intent. If key info is missing, it asks one targeted question; if all fields are known, it executes tool actions directly.

🧠

3. SQLite Persistence Memory

Continuous learning: The agent remembers preferences, prior conversations, and promise-to-pay commitments. Call 2 references prior conversations, bypassing duplicate questions to resolve disputes in record time.

🛡️

Policy Grounded RAG

Every policy statement has strict references. The agent queries corporate knowledge base documents (22 articles covered) and attaches citations directly to the conversation turn.

⛓️

Live CRM & Payments API

Fully integrated with HubSpot CRM and Razorpay API to fetch customer profiles and generate active, test-mode payment links dynamically on waiver validation.

📈

Rigorous Evaluation Harness

The system evaluates intent accuracy, grounding citations, resolution rates, and turns saved on memory recalls automatically via a 64-scenario test harness.
